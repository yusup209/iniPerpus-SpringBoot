1. Pengantar

1.1 Tentang iniPerpus

iniPerpus adalah sistem manajemen perpustakaan modern yang dirancang khusus untuk memudahkan pengelolaan perpustakaan sekolah. Aplikasi ini dilengkapi dengan fitur pengenalan wajah untuk presensi siswa, membuat proses pencatatan kehadiran menjadi lebih cepat dan akurat.

1.2 Fitur Utama

  • Manajemen data siswa lengkap dengan foto wajah
  • Manajemen koleksi buku dengan tracking ketersediaan
  • Sistem peminjaman dan pengembalian buku
  • Presensi otomatis menggunakan teknologi pengenalan wajah
  • Dashboard yang intuitif dan mudah digunakan
  • Sistem keamanan berlapis dengan autentikasi user
  • Tampilan responsif yang dapat diakses dari berbagai perangkat

1.3 Kebutuhan Sistem

Komponen Spesifikasi Minimum
Browser Chrome 90+, Firefox 88+, Edge 90+, Safari 14+
Koneksi Internet Stabil untuk akses aplikasi web
Webcam Diperlukan untuk fitur presensi wajah (min. 720p)
Resolusi Layar Minimum 1024x768 pixels

2. Login dan Autentikasi

2.1 Mengakses Aplikasi

Untuk menggunakan aplikasi iniPerpus, Anda harus melakukan login terlebih dahulu menggunakan akun yang telah terdaftar.

1 Buka browser dan akses URL aplikasi (contoh: http://localhost:8080)
2 Masukkan email dan password Anda pada form login
3 Klik tombol "Sign In" untuk masuk
4 Jika kredensial benar, Anda akan diarahkan ke Dashboard
Catatan: Akun default administrator:
  • Email: admin@example.com
  • Password: adminpass
Segera ganti password default setelah login pertama kali!

2.2 Pesan Error Login

Jika login gagal, sistem akan menampilkan pesan error dengan ikon merah (‚úñ). Beberapa penyebab umum:

  • Email atau password salah
  • Akun belum terdaftar
  • Koneksi ke server bermasalah

2.3 Logout Berhasil

Setelah logout, Anda akan melihat pesan sukses dengan ikon hijau (‚úì) yang menginformasikan bahwa Anda telah keluar dari sistem.

4. Manajemen Siswa

4.1 Mengakses Halaman Siswa

Klik menu "Student Management" dari Dashboard untuk masuk ke halaman manajemen siswa.

4.2 Menambah Siswa Baru

1 Pada bagian "Add New Student", isi form dengan data siswa:
  • Name - Nama lengkap siswa
  • Student ID - Nomor Induk Siswa (NIS)
  • Class - Kelas siswa (contoh: 7A, 8B)
2 Klik tombol "Add User" untuk menyimpan data
3 Siswa baru akan muncul di tabel "Student List"
Tips: Pastikan Student ID unik untuk setiap siswa. Sistem tidak akan menerima Student ID yang sama.

4.3 Melihat Daftar Siswa

Tabel "Student List" menampilkan semua siswa yang terdaftar dengan kolom:

  • Name - Nama siswa
  • Student ID - NIS siswa
  • Class - Kelas siswa
  • Action - Tombol edit (‚úé) dan hapus (üóëÔ∏è)
  • Face - Tombol kamera (üì∏) untuk registrasi wajah

4.4 Mengedit Data Siswa

1 Klik ikon edit (‚úé) pada baris siswa yang ingin diubah
2 Modal dialog "Edit Student" akan muncul dengan data siswa
3 Ubah data yang diperlukan (Name, Student ID, atau Class)
4 Klik tombol "Save" untuk menyimpan perubahan
5 Klik tombol "Cancel" jika ingin membatalkan

4.5 Menghapus Siswa

1 Klik ikon hapus (üóëÔ∏è) pada baris siswa yang ingin dihapus
2 Dialog konfirmasi akan muncul untuk memastikan
3 Klik "Delete" untuk menghapus atau "Cancel" untuk membatalkan
Peringatan: Menghapus siswa akan menghapus semua data terkait termasuk riwayat peminjaman, presensi, dan data wajah. Tindakan ini tidak dapat dibatalkan!

4.6 Paginasi

Jika data siswa banyak, gunakan kontrol paginasi di bawah tabel untuk navigasi:

  • Rows per page - Pilih jumlah data per halaman (5, 10, atau 25)
  • Page indicator - Menunjukkan halaman aktif dan total halaman
  • Prev/Next - Tombol navigasi halaman

4.7 Registrasi Wajah

Klik ikon kamera (üì∏) untuk menuju halaman presensi dan mendaftarkan wajah siswa. Detail lengkap ada di bagian "Presensi Siswa".

5. Manajemen Buku

5.1 Mengakses Halaman Buku

Klik menu "Book Management" dari Dashboard atau navigation bar.

5.2 Menambah Buku Baru

1 Di bagian "Add New Book", isi form dengan informasi buku:
  • Title - Judul buku
  • Author - Nama pengarang
  • ISBN - Nomor ISBN (opsional)
  • Copies - Jumlah eksemplar buku
2 Klik tombol "Add Book" untuk menyimpan
3 Buku baru akan tampil di "Book List"

5.3 Daftar Buku

Tabel "Book List" menampilkan koleksi buku dengan informasi:

  • Title - Judul buku
  • Author - Pengarang
  • ISBN - Nomor ISBN
  • Available - Jumlah eksemplar tersedia / total
  • Action - Tombol edit (‚úé) dan hapus (üóëÔ∏è)
Catatan: Kolom "Available" menunjukkan status ketersediaan. Contoh: "3 / 5" berarti 3 eksemplar tersedia dari total 5 eksemplar.

5.4 Mengedit Buku

1 Klik ikon edit (‚úé) pada buku yang ingin diubah
2 Modal "Edit Book" akan muncul dengan data buku
3 Update informasi yang diperlukan
4 Klik "Save" untuk menyimpan atau "Cancel" untuk batal

5.5 Menghapus Buku

1 Klik ikon hapus (üóëÔ∏è) pada buku yang akan dihapus
2 Konfirmasi penghapusan akan muncul
3 Klik "Delete" untuk konfirmasi atau "Cancel" untuk batal
Peringatan: Buku yang sedang dipinjam tidak dapat dihapus. Pastikan semua peminjaman telah dikembalikan terlebih dahulu.

6. Manajemen Peminjaman

6.1 Mengakses Halaman Peminjaman

Klik menu "Lending Management" dari Dashboard.

6.2 Meminjamkan Buku

1 Di bagian "Lend a Book", pilih siswa dari dropdown "Select Student"
2 Pilih buku yang akan dipinjamkan dari dropdown "Select Book"
3 Pilih tanggal jatuh tempo di field "Due Date"
Catatan: Tanggal jatuh tempo default adalah 14 hari dari tanggal peminjaman. Anda dapat mengubahnya sesuai kebutuhan.
4 Klik tombol "Lend Book" untuk memproses peminjaman
5 Pesan sukses akan muncul jika peminjaman berhasil
Tips: Sistem otomatis akan mengurangi jumlah eksemplar tersedia saat buku dipinjamkan dan menambahnya kembali saat dikembalikan.

6.3 Melihat Daftar Peminjaman

Tabel "Active Loans" menampilkan semua peminjaman yang belum dikembalikan:

  • Student - Nama peminjam
  • Book - Judul buku yang dipinjam
  • Lent Date - Tanggal peminjaman
  • Due Date - Tanggal jatuh tempo
  • Action - Tombol "Return" untuk mengembalikan

6.4 Mengembalikan Buku

1 Cari peminjaman yang akan dikembalikan di tabel "Active Loans"
2 Klik tombol "Return" pada baris peminjaman tersebut
3 Peminjaman akan dihapus dari daftar dan stok buku otomatis bertambah

6.5 Status dan Notifikasi

Di bagian atas form peminjaman, sistem menampilkan pesan status:

  • Hijau - Peminjaman atau pengembalian berhasil
  • Merah - Error (buku tidak tersedia, data tidak valid, dll)

7. Presensi Siswa (Pengenalan Wajah)

7.1 Mengakses Halaman Presensi

Klik menu "Student Presence" dari Dashboard atau ikon kamera (üì∏) dari halaman siswa.

[Gambar: Halaman Presensi Siswa]

7.2 Memulai Kamera

1 Klik tombol "Start Camera" untuk mengaktifkan webcam
2 Browser akan meminta izin akses kamera - klik "Allow"
[Gambar: Pop-up Izin Akses Kamera]
3 Video feed dari kamera akan muncul di layar
4 Tombol berubah menjadi "Stop Camera" untuk menonaktifkan kamera
[Gambar: Kamera Aktif Menampilkan Video Feed]
Catatan: Pastikan pencahayaan ruangan cukup dan wajah terlihat jelas oleh kamera untuk hasil terbaik.

7.3 Mendaftarkan Wajah Siswa

Sebelum siswa dapat melakukan check-in otomatis, wajah mereka harus didaftarkan terlebih dahulu.

1 Pastikan kamera sudah aktif
2 Pilih nama siswa dari dropdown "Select Student to Register"
[Gambar: Dropdown Pilih Siswa untuk Registrasi]
3 Posisikan wajah siswa di depan kamera:
  • Wajah menghadap lurus ke kamera
  • Jarak 30-50 cm dari kamera
  • Pencahayaan merata tanpa bayangan
  • Ekspresi wajah netral
[Gambar: Contoh Posisi Wajah yang Benar]
4 Klik tombol "Register Face"
5 Sistem akan memproses dan menampilkan pesan:
  • Biru - "‚úì Face Registered" (berhasil)
  • Merah - Error jika gagal mendeteksi wajah
[Gambar: Pesan Registrasi Wajah Berhasil]
Tips: Untuk akurasi terbaik, daftarkan wajah di kondisi pencahayaan yang mirip dengan kondisi saat check-in nantinya.

7.4 Check-In Otomatis

Setelah wajah terdaftar, siswa dapat melakukan check-in presensi secara otomatis.

1 Pastikan kamera aktif
2 Posisikan wajah siswa di depan kamera
3 Klik tombol "Check In"
4 Sistem akan mencocokkan wajah dengan database:
  • Hijau - "‚úì Check-In Successful!" jika wajah dikenali
  • Merah - "‚úó Face Not Recognized" jika tidak cocok
[Gambar: Pesan Check-In Berhasil]
[Gambar: Pesan Check-In Gagal]

7.5 Melihat Riwayat Check-In

Tabel "Recent Check-Ins" menampilkan presensi terbaru dengan informasi:

  • Student - Nama siswa
  • Check-In Time - Waktu presensi
  • Match Status - Status pencocokan (Matched / Unmatched)
[Gambar: Tabel Riwayat Check-In]

7.6 Update Wajah (Re-register)

Jika wajah siswa berubah (tumbuh dewasa, ganti gaya rambut, dll) atau sistem sering gagal mengenali, lakukan registrasi ulang:

1 Pilih siswa yang sama dari dropdown
2 Klik "Register Face" lagi
3 Data wajah lama akan diganti dengan yang baru
Catatan: Re-register tidak menghapus riwayat check-in sebelumnya.

Masalah Solusi

9. Troubleshooting

9.1 Masalah Umum dan Solusi

Tidak Bisa Login

Gejala: Pesan error "Invalid username or password"

Solusi:

  • Pastikan email dan password yang dimasukkan benar
  • Periksa capslock tidak aktif
  • Coba reset password atau hubungi administrator

Halaman Tidak Muncul / Blank

Gejala: Layar kosong atau tidak ada data yang tampil

Solusi:

  • Refresh browser (F5 atau Ctrl+R)
  • Clear cache dan cookies browser
  • Coba browser lain (Chrome, Firefox, Edge)
  • Periksa koneksi internet

Data Tidak Tersimpan

Gejala: Form berhasil disubmit tapi data tidak muncul di tabel

Solusi:

  • Periksa apakah ada pesan error
  • Pastikan semua field required terisi
  • Refresh halaman
  • Logout dan login kembali

Kamera Tidak Berfungsi

Gejala: Video feed tidak muncul saat klik "Start Camera"

Solusi:

  • Periksa izin kamera di browser (klik ikon gembok di address bar)
  • Pastikan webcam terhubung dan driver terinstall
  • Tutup aplikasi lain yang menggunakan kamera (Zoom, Skype, dll)
  • Restart browser
  • Coba browser lain

Wajah Tidak Dikenali

Gejala: Check-in selalu gagal meski wajah sudah terdaftar

Solusi:

  • Perbaiki pencahayaan ruangan
  • Pastikan wajah menghadap lurus ke kamera
  • Hapus kacamata atau aksesoris yang menutupi wajah
  • Lakukan re-register wajah
  • Pastikan Python face service berjalan

Aplikasi Lambat

Gejala: Loading lama atau aplikasi terasa lag

Solusi:

  • Periksa koneksi internet
  • Tutup tab browser lain yang tidak perlu
  • Clear cache browser
  • Restart browser
  • Hubungi administrator jika masalah berlanjut

9.2 Menghubungi Dukungan Teknis

Jika masalah tidak dapat diselesaikan dengan panduan di atas, hubungi administrator sistem dengan informasi berikut:

  • Deskripsi masalah yang detail
  • Browser dan versi yang digunakan
  • Screenshot error jika ada
  • Langkah-langkah yang sudah dicoba

10. FAQ (Pertanyaan Umum)

Q: Apakah satu siswa bisa meminjam lebih dari satu buku?

A: Ya, siswa dapat meminjam beberapa buku sekaligus. Setiap peminjaman akan tercatat terpisah di tabel Active Loans.

Q: Berapa lama periode peminjaman buku?

A: Default periode peminjaman adalah 14 hari. Administrator dapat mengubah tanggal jatuh tempo sesuai kebutuhan saat meminjamkan buku.

Q: Apakah bisa menghapus data peminjaman?

A: Peminjaman yang sudah dikembalikan akan otomatis terhapus dari Active Loans. Untuk membatalkan peminjaman yang belum dikembalikan, gunakan tombol Return.

Q: Apakah data wajah aman?

A: Ya, data wajah disimpan dalam bentuk encoding matematis (bukan foto) di server yang aman. Data hanya digunakan untuk presensi dan tidak disebarkan.

Q: Bagaimana jika siswa lupa check-in?

A: Administrator dapat melihat riwayat check-in di tabel Recent Check-Ins. Untuk koreksi manual, hubungi administrator sistem.

Q: Apakah bisa akses dari smartphone?

A: Ya, aplikasi responsif dan dapat diakses dari smartphone. Namun fitur pengenalan wajah akan lebih optimal menggunakan webcam komputer/laptop.

Q: Berapa maksimal jumlah siswa yang bisa terdaftar?

A: Tidak ada batasan hard-coded. Kapasitas tergantung spesifikasi server yang digunakan.

Q: Apakah ada laporan atau statistik?

A: Saat ini aplikasi menampilkan total count untuk siswa, buku, peminjaman, dan presensi. Fitur laporan detail sedang dalam pengembangan.

Q: Bagaimana cara backup data?

A: Backup data adalah tanggung jawab administrator. Hubungi administrator untuk informasi jadwal backup dan restore data.

Q: Apakah bisa custom tampilan atau fitur?

A: Ya, aplikasi ini open-source dan dapat dikustomisasi. Hubungi developer atau administrator untuk request fitur khusus.
